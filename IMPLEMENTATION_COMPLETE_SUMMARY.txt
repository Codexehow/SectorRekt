PROBLEM IDENTIFIED
================================================================================

Issue: OverHeat UI was not updating properly (jerky animation, poor performance)

Root Cause: Every frame, the signal handler was creating a NEW StyleBoxFlat 
object instead of reusing one. This caused:
  - 60 StyleBoxFlat allocations per second
  - 60 theme system updates per second
  - Jerky animation (visible stuttering)
  - Memory waste and CPU overhead
  - Unprofessional appearance

================================================================================
SOLUTION IMPLEMENTED
================================================================================

Strategy: Cache the StyleBox once and just update its color each frame

File Modified: res://ui/cpu_hud.gd

Change 1 - Line 21: Add cache variable
────────────────────────────────────────
var overheat_fill_stylebox: StyleBoxFlat = null  # Cache StyleBox

Change 2 - Lines 63-66: Initialize in _ready()
────────────────────────────────────────────────
# Create the fill StyleBox once to avoid recreating it every frame
overheat_fill_stylebox = StyleBoxFlat.new()
overheat_fill_stylebox.bg_color = Color.YELLOW
overheat_bar.add_theme_stylebox_override("fill", overheat_fill_stylebox)

Change 3 - Line 206: Update function logic
──────────────────────────────────────────
# OLD: var fill_stylebox := StyleBoxFlat.new()
#      fill_stylebox.bg_color = overheat_color
#      overheat_bar.add_theme_stylebox_override("fill", fill_stylebox)
# 
# NEW: Just update the cached color
overheat_fill_stylebox.bg_color = Color.YELLOW.lerp(Color.RED, color_ratio)
