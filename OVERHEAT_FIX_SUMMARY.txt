===============================================================================
                        OVERHEAT SYSTEM - FIX SUMMARY
===============================================================================

ISSUE: Overheat bar remains stuck at 0% no matter how many times player 
       right-clicks to generate CPU cycles. Player never sees heat increase.

ROOT CAUSE: The overheat trigger condition checks if CPU >= 100.0, but due to
            constant CPU decay in every frame, CPU can never reach exactly 100%.
            CPU oscillates around 97-99%, so the condition is ALWAYS FALSE.

LOCATION: res://player/player.gd, lines 242-256 (generate_cpu_cycles function)

===============================================================================
                              THE FIX
===============================================================================

BEFORE (BROKEN):
  var was_at_max: bool = current_cpu >= max_cpu_cycles  # Checks >= 100.0
  if was_at_max and current_cpu >= max_cpu_cycles:      # Never true!

AFTER (FIXED):
  var cpu_at_max_threshold: float = max_cpu_cycles * 0.95  # Use 95%
  var was_at_max: bool = current_cpu >= cpu_at_max_threshold
  if was_at_max and current_cpu >= cpu_at_max_threshold:    # Now works!

WHY 95%?
  - CPU naturally reaches 95-99% when player holds right-click
  - This is a reasonable "effectively at max" threshold
  - Allows for floating-point precision and decay mechanics
  - Still enforces the anti-spam penalty as designed
