USER REPORTED ISSUE:
  "When I click right-mouse button to generate CPU cycles, no matter how
   many times I click, the Overheat bar never exceeds 0%. The CPU bar gets
   to 97-99% but heat is never generated."

INVESTIGATION RESULTS:
  ✓ Root cause identified: CPU threshold check impossible to reach
  ✓ CPU decays every frame, preventing exact 100% from ever occurring
  ✓ Old code checked: if (cpu >= 100.0) → Always FALSE
  ✓ Result: Overheat system completely non-functional

═══════════════════════════════════════════════════════════════════════════

THE FIX APPLIED:

  FILE: res://player/player.gd
  FUNCTION: generate_cpu_cycles()
  LINES: 242-256
  
  BEFORE (BROKEN):
  ──────────────────────────────────────────────────────────────────────────
  var was_at_max: bool = current_cpu >= max_cpu_cycles
  if was_at_max and current_cpu >= max_cpu_cycles:
      overheat += heat_from_click
  ──────────────────────────────────────────────────────────────────────────
  Problem: Checks if CPU >= 100.0 (never true due to decay)
  Result: Overheat NEVER increases
  
  AFTER (FIXED):
  ──────────────────────────────────────────────────────────────────────────
  var cpu_at_max_threshold: float = max_cpu_cycles * 0.95
  var was_at_max: bool = current_cpu >= cpu_at_max_threshold
  if was_at_max and current_cpu >= cpu_at_max_threshold:
      overheat += heat_from_click
  ──────────────────────────────────────────────────────────────────────────
  Solution: Checks if CPU >= 95.0 (always true when CPU is 97-99%)
  Result: Overheat increases correctly

═══════════════════════════════════════════════════════════════════════════

WHY 95% WORKS:

  1. CPU naturally reaches 95-99% when held at high usage
  2. Game mechanics prevent exact 100% due to frame-by-frame decay
  3. 95% represents "effectively at maximum" for practical purposes
  4. Provides safe margin for floating-point precision
  5. Maintains original game design intent (anti-spam mechanic)
