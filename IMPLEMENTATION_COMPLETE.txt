                              FEATURE #1: CPU DRAIN SYSTEM
================================================================================

DESCRIPTION:
When CPU cycles drop to 0, all systems begin losing power rapidly. This creates
a critical resource management challenge - players must constantly click to
maintain their capabilities or face complete system failure.

IMPLEMENTATION:
File: res://player/player.gd (lines 109-115)

When current_cpu <= 0:
  - Weapon Charge: -30.0 points/second
  - Shield Charge: -30.0 points/second
  - Blink Charge: -30.0 points/second
  - Movement Speed: -50.0 units/second (minimum 30 units/second)

GAMEPLAY IMPACT: ⭐⭐⭐⭐⭐ (Very High)
- Makes CPU management critical to survival
- Forces continuous player engagement
- Creates difficulty spike when neglecting click generation

================================================================================
                           FEATURE #2: SHIELD BUFFER SYSTEM
================================================================================

DESCRIPTION:
A sophisticated two-phase shield recharge mechanic that prevents instant shield
recovery and creates risk/reward gameplay:

PHASE 1 - ACCUMULATION (Shields at 100%):
  - CPU allocation to shields goes into a buffer instead of shields
  - Buffer holds up to 500 points maximum
  - Accumulates when shields are full at: CPU * 0.30 * 2.5 * delta_time

PHASE 2 - RECOVERY (Shields below 10%):
  - Buffer slowly recharges shields at 5.0 HP/second
  - Never instant - creates tension and difficulty
  - Player must survive with low shields while buffer recovers them

IMPLEMENTATION:
Files: 
  - res://player/player.gd (lines 41-46, 117-138, 185)
  - res://ui/cpu_hud.gd (lines 97-98)
  - res://ui/cpu_hud.tscn (lines 133-137)

Variables:
  var shield_buffer: float = 0.0
  var shield_buffer_recharge_rate: float = 5.0
  var shield_low_threshold: float = 10.0
  signal shield_buffer_updated(buffer: float)

UI DISPLAY:
  - New "Buffer: X" label shows current buffer value
  - Located under Shield bar in the HUD
  - Color: Light blue (0, 0.5, 1, 1)

GAMEPLAY IMPACT: ⭐⭐⭐⭐ (Very High)
- Shields can't instantly recover, creating extended vulnerability
- Players must manage both shield health and buffer reserves
- Adds strategic depth to shield management
- Makes the game significantly more difficult

EXAMPLE SCENARIO:
1. Player generates CPU while shields full → buffer accumulates
2. Enemy hits player → shields drop from 100% to 8%
3. Buffer doesn't activate yet (shields still above 10%)
4. Player survives a few more hits
5. Shields drop below 10% → buffer starts recharging at 5 HP/sec
6. Player waits 20 seconds to recover 100 shields (if buffer had 100 stored)
