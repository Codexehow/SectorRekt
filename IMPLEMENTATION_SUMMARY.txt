OVERVIEW
════════════════════════════════════════════════════════════════════════════════

When the Overheat meter reaches 100%, the Consequence Engine triggers:
  1. Game PAUSES (get_tree().paused = true)
  2. Popup appears with dramatic digital/corruption theme
  3. Player chooses one of two negative consequences
  4. Consequence is applied to the tank
  5. Game RESUMES with overheat reset to 0%

════════════════════════════════════════════════════════════════════════════════

File: res://ui/consequence_engine.gd (86 lines)

Function: Manages the entire consequence flow
  1. Listens for overheat_critical signal
  2. Pauses the game
  3. Creates and shows popup
  4. Waits for player choice
  5. Applies consequence
  6. Resets overheat
  7. Unpauses game

Methods:
  • _ready() - Connect to player signals
  • _on_overheat_critical() - Pause and show popup
  • show_consequence_popup() - Create popup UI
  • _on_consequence_selected() - Apply consequence

Signal Connection (in _ready):

GAMEPLAY INSTRUCTIONS
════════════════════════════════════════════════════════════════════════════════

To Trigger Consequence Engine:

1. Generate CPU
   └─ Right-Click (mouse) or Q (keyboard)
   └─ Each click adds 25 CPU, costs 12.5 heat when at max

2. Build Overheat
   └─ Keep generating CPU when already at 95%+
   └─ Overheat bar fills: Yellow → Orange → Red
   └─ Reaches 100% after ~23-25 clicks at max CPU

3. Consequence Popup Appears
   └─ Game pauses
   └─ Popup shows with two buttons
   └─ Read consequence descriptions

4. Choose Consequence
   └─ Click "Movement Lockdown" or "Blink Drive Reset"
   └─ Game unpauses immediately
   └─ Consequence applied
   └─ Overheat reset to 0%

5. Recover
   └─ Deal with consequence penalty
   └─ Continue playing while recovering
   └─ Can trigger consequences again if overheat builds
