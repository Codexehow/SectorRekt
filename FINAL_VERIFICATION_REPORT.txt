ISSUE DETAILS
═══════════════════════════════════════════════════════════════════════════════

Error Message:
  "Invalid call. Nonexistent function 'set_border_enabled' in base 'StyleBoxFlat'."

Trigger:
  When consequence system attempts to display consequence selection UI popup
  during game overheat sequence

Impact:
  ❌ Consequence popup cannot be displayed
  ❌ Game cannot proceed after tank overheat
  ❌ Player forced to quit on overheat


ROOT CAUSE ANALYSIS
═══════════════════════════════════════════════════════════════════════════════

Problem: Using Godot 3.x API in Godot 4 project

Godot 3.x (Deprecated):
  var style: StyleBoxFlat = StyleBoxFlat.new()
  style.set_border_enabled(true)      ← Method exists in Godot 3
  style.set_border_width_all(2)

Godot 4.x (Current):
  var style: StyleBoxFlat = StyleBoxFlat.new()
  style.set_border_width_all(2)       ← set_border_enabled() was REMOVED
  style.border_color = color

Why: Godot 4 simplified the API - borders are automatically visible when width > 0

Correct StyleBoxFlat Usage in Godot 4.x:

  # Create StyleBoxFlat
  var style: StyleBoxFlat = StyleBoxFlat.new()

  # Set background color
  style.bg_color = Color(0.1, 0.1, 0.2, 0.9)

  # Set border (BOTH required)
  style.border_color = Color(1.0, 0.0, 1.0, 1.0)
  style.set_border_width_all(2)  # or set individual borders

  # Optional: Individual border control
  # style.border_width_left = 2

  # style.border_width_right = 2
  # style.border_width_bottom = 1

  # NEVER do this in Godot 4 (causes crash):
